# 掷飞刀（Knife Hit）游戏开放文档

基于 **Cocos Creator 3.8** 的轻量级点击反应类休闲游戏。玩家点击屏幕向上掷出飞刀，命中轮盘并附着，累计达到目标数量即胜利；若与已附着飞刀相撞则失败。

---

## 1. 项目概述

### 1.1 核心体验

- 节奏短平快，单局时长短；
- 操作简单（单击即可）；
- 难点集中在时机判断与多轮盘、移动轨迹下的节奏控制。

### 1.2 脚本结构

| 脚本 | 挂载位置 | 职责 |
|------|----------|------|
| `Game.ts` | 游戏根节点 | 回合流程、输入、飞刀生成/飞行、胜负结算 |
| `WheelController.ts` | 各轮盘节点（`gp_wheel` 子节点） | 单轮自转、碰撞、飞刀附着、命中抖动、失败特写 |
| `WheelGroupPathController.ts` | `gp_wheel`（运行时自动添加） | 轮盘组沿路径贪吃蛇式移动 |

---

## 2. 场景节点结构

推荐层级如下（`main.scene` 已按此组织）：

```
Canvas
├── Game                    # 挂 Game.ts
├── gp_wheel                # wheelsRoot：轮盘容器
│   ├── wheel               # 挂 WheelController（蛇头，sibling 第一个）
│   └── wheel-001           # 挂 WheelController（蛇身）
├── gp_path                 # pathRoot：轨迹路点（子节点顺序即路径）
│   ├── Sprite              # 路点 1（运行时自动移除 Sprite 组件）
│   ├── Sprite-001          # 路点 2
│   └── ...
├── slot_knife              # knifeSpawnPoint：飞刀出生点
└── UI（进度、胜利/失败面板等）
```

### 2.1 单轮盘内部层级（`WheelController` 自动维护）

```
wheel（outer）
└── focus                   # 失败特写缩放层
    └── middle              # 自转 + 命中上下抖动 + 已附着飞刀
        └── body            # 轮盘视觉 + CircleCollider2D
```

## 3. 核心玩法规则

### 3.1 场景布局

1. **轮盘组**位于 `gp_wheel` 下，可配置多颗轮盘；
2. 轮盘组整体沿 **`gp_path` 路点折线** 做贪吃蛇式移动（第一个子轮盘为蛇头，其余按间距跟随）；
3. 屏幕下方 **`slot_knife`** 为待发飞刀出生点。

### 3.2 操作方式

1. 玩家点击屏幕；
2. 当前待发飞刀沿世界 Y 轴正方向直线上升；
3. 飞刀命中**任意一颗**轮盘即有效得分，附着到该轮盘并随其自转；
4. 命中后生成下一把待发飞刀（受 `knifeSpawnInterval` 间隔限制）。

### 3.3 胜负规则

| 结果 | 条件 |
|------|------|
| **胜利** | 累计命中飞刀数 ≥ `targetKnifeCount` |
| **失败** | 飞行中的飞刀与**任意轮盘**上已附着飞刀发生碰撞 |
| **无惩罚** | 飞刀未命中任何轮盘并飞出屏幕，直接销毁 |

- 胜利 / 失败后弹出对应面板，点击「继续游戏」开始新一局；
- 撞飞刀失败时，先播放**慢镜头特写**（镜头对准碰撞点并放大 + 红色闪烁），再显示失败面板。

---

## 4. 一局完整流程

1. 初始化：清理飞刀、重置各轮盘、轮盘组回到路径起点；
2. 轮盘组沿 `gp_path` 移动，各轮盘自转（可受飞刀失衡影响变速）；
3. 显示待发飞刀，等待点击；
4. 飞刀上升，遍历所有轮盘做碰撞检测：
   - **优先**判定是否撞到已附着飞刀 → 失败特写；
   - 否则判定是否命中轮盘圆碰撞体 → 附着、计数 +1、播放命中抖动；
5. 达到目标数 → 胜利；否则继续；
6. 结算界面点击「继续游戏」→ 回到步骤 1。

---

## 5. 关键系统设计

### 5.1 轮盘组轨迹（贪吃蛇）

由 `WheelGroupPathController` 驱动：

- **`gp_path` 子节点顺序** 定义折线路径（至少 2 个路点）；
- **`gp_wheel` 子节点顺序** 定义蛇头 → 蛇身（第一个子节点为蛇头）；
- 场景加载后自动 **移除路点上的 `Sprite` 组件**（仅作编辑器标记，运行时不可见）；
- 失败特写前，将发生碰撞的轮盘 **提到 sibling 队尾**，避免被其它轮盘遮挡；新一局恢复原始顺序。

### 5.2 单轮盘系统（`WheelController`）

- 中间层 `middle` 负责**自转**与**命中上下抖动**；
- 特写层 `focus` 负责失败时的**镜头推近 + 缩放**；
- 支持 **飞刀失衡重力**：附着飞刀分布不均时，轮盘转速会周期性加减速；
- 飞刀命中后：剑尖落在轮盘圆周交点，播放**立正旋转**动画（`attachedKnifeAlignDuration`）。

### 5.3 碰撞判定

| 检测类型 | 方法 | 说明 |
|----------|------|------|
| 飞刀 vs 轮盘 | AABB + 圆/多边形采样 | 轮盘使用 `CircleCollider2D` |
| 飞刀 vs 已附着飞刀 | SAT 多边形重叠 | 飞刀使用 `PolygonCollider2D` |

判定优先级（每帧、每把飞行飞刀）：

1. 遍历所有轮盘 → 是否撞到已附着飞刀；
2. 遍历所有轮盘 → 是否命中轮盘本体。

### 5.4 失败特写

触发条件：飞行飞刀与已附着飞刀碰撞。

流程：

1. 暂停轮盘组轨迹移动；
2. 将事故轮盘提至渲染队尾；
3. 以两把飞刀 **碰撞交集中心** 为锚点，同步平移 + 放大，使焦点移至屏幕中心（`failFocusMoveDuration`）；
4. 碰撞飞刀红色闪烁（`failFocusDuration` / `failKnifeBlinkInterval`）；
5. 显示失败面板。

### 5.5 游戏状态机

```
Idle → KnifeFlying → Idle（继续本局）
                   → RoundWin / RoundFail → 继续游戏 → 新一局
```

- `Idle` / `KnifeFlying`：驱动轨迹移动与轮盘自转；
- `RoundWin` / `RoundFail`：冻结移动，播放结算与特写；
- 输入在 `inputLockedUntil` 之前被屏蔽，避免点击「继续游戏」误触发射。

## 6. 编辑器配置

### 6.1 Game 组件必绑字段

| 属性 | 节点 |
|------|------|
| `wheelsRoot` | `gp_wheel` |
| `pathRoot` | `gp_path` |
| `knifeSpawnPoint` | `slot_knife` |
| `knifePrefab` | 飞刀预制体 |

### 6.2 添加新轮盘

1. 在 `gp_wheel` 下复制轮盘节点；
2. 调整位置（运行后由轨迹控制器覆盖）；
3. 配置该节点上的 `WheelController` 参数（转速、抖动等）；
4. **Sibling 顺序**：越靠前越靠近蛇头。

### 6.3 编辑移动路径

1. 在 `gp_path` 下添加/移动路点子节点；
2. 子节点 **Sibling 顺序** 即为路径走向；
3. 路点可挂 `Sprite` 便于编辑器可视化，运行后自动移除。

### 6.4 飞刀预制体要求

- 锚点建议在剑尖（如 `(0.5, 1)`）；
- 需挂 **`PolygonCollider2D`**，轮廓与飞刀外形一致。

## 7. 关键参数

### Game.ts

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `targetKnifeCount` | 8 | 本局目标命中数 |
| `knifeFlySpeed` | 1200 | 飞刀上升速度（像素/秒） |
| `knifeSpawnInterval` | 0.2 | 连续发射最小间隔（秒） |
| `failFocusDuration` | 2 | 失败特写 + 闪烁总时长 |
| `failFocusScale` | 1.12 | 失败特写放大倍数 |
| `failKnifeBlinkInterval` | 0.15 | 红色闪烁切换间隔 |

### WheelGroupPathController.ts

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `moveSpeed` | 140 | 蛇头沿轨迹速度（像素/秒） |
| `snakeSpacing` | 200 | 相邻轮盘轨迹间距 |
| `loopPath` | true | 路径是否首尾闭合 |

### WheelController.ts（每轮盘独立）

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `wheelRotateSpeed` | 120 | 基础自转速度（度/秒） |
| `wheelImbalanceEnabled` | true | 是否启用失衡重力变速 |
| `wheelHitShakeDistance` | 10 | 命中上下抖动幅度 |
| `attachedKnifeAlignDuration` | 0.2 | 飞刀立正动画时长 |
| `failFocusMoveDuration` | 0.35 | 失败镜头推近时长 |

## 8. 开发验收清单

- [ ] `gp_wheel` 下多颗轮盘沿 `gp_path` 贪吃蛇式移动；
- [ ] 路点 Sprite 运行后不可见，路径逻辑正常；
- [ ] 飞刀从下方出生点发射并直线上升；
- [ ] 命中**任一轮盘**均可附着、计数 +1；
- [ ] 飞刀随轮盘自转，命中时有抖动反馈；
- [ ] 达到目标数量触发胜利面板；
- [ ] 撞到已附着飞刀触发失败特写（不遮挡、焦点居中）；
- [ ] 飞出屏幕未命中不判失败；
- [ ] 点击「继续游戏」正确重开，轮盘组回到路径起点。
